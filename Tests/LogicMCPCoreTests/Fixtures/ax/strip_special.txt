# ── BUG 1: the output-routing slot must be found POSITIVELY ────────────────────────────────
# The routing slot is the IMMEDIATE NEXT SIBLING of the `AXPopUpButton description="group"`,
# and only if it is a PLAIN AXButton (no subrole, no value) whose description is not one of the
# fixed control labels. Its description IS the destination ("Bus 21", "Stereo Output") — which
# is why the old by-EXCLUSION search (first AXButton whose description isn't a known label)
# looked plausible, and why it is WRONG:
#
#   * Stereo Out has NO routing slot at all → by-exclusion matched `bounce` (an AXSwitch).
#   * Master    has NO routing slot at all → by-exclusion matched `dim`    (an AXSwitch).
#
# Both were reported live as the strip's `output` in refresh_state. Do NOT pattern-match the
# description against bus-name shapes (`Bus \d+`) either — buses are RENAMABLE in Logic.
#
# Fixed control labels seen on strips (the negative guard; gathered from this capture and
# Fixtures/ax/mixer_strip.txt): name, mute, solo, volume fader, fader knob, volume fader level,
# peak level meter, pan, knob readout, automation, list, group, send button, audio plug-in,
# MIDI plug-in, insert bar, EQ, gain reduction meter, setting, channel mode, bypass, open,
# bounce, dim, mastering assistant, add aux, record.
#
# ── BUG 2: BOTH windows contain an `AXLayoutArea description="Mixer"` ──────────────────────
# The arrange window's INSPECTOR holds a MINI-MIXER (the selected track + its output, 2 strips)
# with the SAME role+description as the real thing. With the Mixer WINDOW closed it is the only
# one left, and binding it silently REPLACES the shadow model with 2 strips while reporting
# success. Only bind a mixer area whose containing WINDOW's title contains "Mixer".
